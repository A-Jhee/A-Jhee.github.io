4 important lessons I learned from building my project:
GECKO bug tracker.

Human Legibility

Code is to be read by humans first, executed by machines second. I focus on writing human legible code for ease of debugging, maintaining, and collaborating.

Expect Bugs

Bugs are nearly guaranteed and always cause headaches. I became comfortable with errors and bugs. This allows me to debug much more effectively.

Always Write Tests

A code that worked a minute ago, can suddenly raise errors after even the smallest change. To prevent regression, I always build a test suite for my code.

Patience & Attention to Detail

Being a successful programmer is less (than I thought) about technical knowledge and more about patient temperment with a fanatical attention to detail.
